Zen Release Notes

1.3.4

Minor UI tweaks. No prefs reset required.

1.3.2

System > Reset Preferences is required for this update.
  • Mesh Fusion improvements: Added Mesh Fusion cut/copy/paste/split commands, and made Fusion tools available from within the Zen context tools.
  • Right-handed prefs separated: The right-handed setting is now separate for toolboxes and lists, allowing you to put lists on one side and toolboxes on the other.
  • Viewport improvements: UV viewport no longer shows UVs from inactive meshes by default. "glmeter" is now available in viewport properties panel.
  • Layout improvements: Channels list now defaults to show all channels. Images & Clips viewer now defaults to List mode. Added "maximize" behavior to Big Preview viewport for GL camera viewport. Removed "recent" tools from header by default (use Zen UI prefs to enable). Added Python interpreter to scripting layout.
  • Preferences improvements: Cleaned up preferences form. Middle-mouse properties popover now works in the Groups panel.

1.3.1

  • Render toolbox: Mini toolbox at left for use with the new big progressive viewport for working with progressive rendering: render settings, camera settings, save image, save layered, pass group/pass, and bucket render.
  • hide/show mini toolbox from toolboxes menu: The toolboxes menu now has a check box for hiding and showing the mini toolbox on the left
  • Third party toolbox support: The toolbox system can now be extended by third parties to add new toolboxes to both the 'V' key and mini toolbars. See developer documentation for instructions.
  • viewport switcher button highlighting: The viewport buttons at bottom center now highlight to indicate which one is currently active
  • New viewports: There are now four viewport options: model, topo, animate, and render. Swtiching between viewports using the buttons at the bottom center will also change the current toolbar: context tools for the model viewport, retopology tools for the topo viewport, animation tools for the animation viewport, and render commands for the progressive render viewport.
  • Quad View Everywhere: Quad view is now supported with the model, topo, and animate viewports.
  • Moved Help menu to the proper place: Zen Help is now under Help > Kit Help
  • Merge vertices from any selection mode: Merge vertices is now available in the "mesh edit" menu from all selection modes.
  • `Enter` to rename in Groups: Enter key now renames in Groups panel like everywhere else
  • Added MODO 11 selection sets UI: Added the new selection sets menu from the MODO 11 modes bar
  • Channels List: Channels list in palettes menu now has "show all" enabled by default
  • Better viewport properties: The viewport properties popover has been completely reworked, and is now mapped to the 'o' key in the Zen default hotkeys. It includes everything the default 'o' key viewport properties has and more, in a much cleaner package.
  • Viewport presets: Viewport presets are now available from the new viewport properties popover
  • Event log: Palettes list now includes the Event Log for developers
  • Better default viewport settings: The modeling, retopology, animation, and camera viewports now have much more sensible default visibility settings.
  • Fixed various bugs: Lots of minor bug fixes

10.8.0

  • Lots more toolboxes: The 'V' key toolbox and the 'mini' toolbox on the left side are now always in sync, showing the toolbox specified in the "Toolbox" popup. The popup is available at the bottom left of Zen in the footer bar, or at the top of the 'V' key toolbox just below the tool modes icons. We've added tons of previously-missing toolbars (hair, particle paint, vertex maps, command regions, inverse kinematics, and tons of others).
  • Smarter, more consistent toolboxes: In addition to adding new toolboxes, the existing ones have been cleaned up and streamlined, removing redundancy and putting things in more predictable places. This is an ongoing effort. Let us know if anything could be better.
  • Layout improvements: The Zen layout has been updated and improved, adding an item properties and channels list to the sidebar, separating the dopesheet from the other frames on the lower half (for independent sizing and simultaneous use), and several other changes necessary for the new retopology improvements (see below).
  • Retopology improvements: There is now a dedicated retopology toolbox in the 'V' key and 'mini' toolbars, and a separate viewport for retopology vs normal modeling. We tried very hard to make it possible to both model and retopo in a single viewport, but it simply wasn't feasible with the MODO viewport system. The new solution is much more consistent, and not too much hassle.
  • Quad modeling viewport: The default modeling viewport is now a maximized quad-modeling viewport. To work in quad mode, click the yellow 'maximize' button at the top right of the modeling viewport. You can, of course, maximize it again the same way. This one was acctually added in 10.7.0, but we forgot to announce it.
  • Bug fixes: Cameras and lights are now visible by default in the model viewport. The graph editor works better and more consistently.

10.7.0

  • Hotkey Mapping Options: Preferences > Mechanical Color > Zen UI > Map Default Zen Hotkeys... Now provides a list of checkboxes for each hotkey. You can choose which to map and which to ignore.
  • Hotkey Defaults: 'V' and 'G' keys are no longer mapped by default. You will need to add these manually using the Zen UI preferences, either using the newly-improved "Map Default Zen Hotkeys..." utility or by manually setting hotkeys using the individual form mapping buttons below.
  • Revert Hotkeys: Hotkey preferences can now be reverted back to their factory defaults (rather than simply clearing them, as before). Recommended hotkey suggestions are included in button labels.
  • Right-handed layout option: Lists and toolboxes can now be displayed on the right using the "Right-Handed List Buttons" toggle in the Zen UI preferences.
  • Toolbox Updates: Toolbox now includes "Poly Reduction" and "Add Point" tools where appropriate.
  • New/Updated install alerts: When new versions of Zen are installed or updated, an alert will fire to remind you to reset your MODO preferences. Afterwards, the release notes will be opened in a new browser window.
  • Bug fix: Topo Mode viewport shading: Viewport shading modes now remember their state when toggling topo mode. This fixes various viewport shading related issues related to zen topo mode.
  • Bug fix: Performance: A significant lag when RMB clicking in extremely large scene files is fixed. You may now right-click with impunity.
